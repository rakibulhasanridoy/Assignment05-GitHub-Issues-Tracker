# 🐙 GitHub Issues Tracker
A dynamic, API-driven dashboard designed to monitor and manage GitHub issues with a secure authentication layer.

---

# 📌 Description
GitHub Issues Tracker is a sophisticated web application that interfaces directly with external APIs to fetch real-time data from GitHub. Built with a focus on asynchronous JavaScript and DOM manipulation, it features a secure login gateway and an interactive dashboard for issue management.

The platform allows developers to filter through various issues states (Open vs. Closed) and perform deep-dives into specific issue data via a dedicated single-card view.

---

# 🛠️ The Tech Stack
- 🏗️ HTML5 (Structure)

- 🎨 Tailwind CSS (Modern Styling & Layout)

- 🧠 JavaScript (API & DOM) (Fetch API, Async/Await)

- 🔐 Authentication Logic (Login Session Management)

---

# ✨ Core Engineering Features
### 🔑 Secure Login Gateway
- Protected Routing: Prevents unauthorized access to the tracker dashboard.

- Session Simulation: Validates user credentials before granting access to the main interface.

### 🔍 Real-Time API Integration
- Dynamic Filtering: Toggle between All, Open, and Closed issues, with data fetched instantly from the API.

- Smart Search: Integrated search functionality that queries the API to find specific issues by title or keyword.

### 📑 Detailed Insight View
- Single-Card Expansion: Clicking on an issue card triggers a detailed view, fetching specific metadata directly from the source.

- State-Aware UI: Automatically reflects the current status and labels of each issue as defined in the API.
